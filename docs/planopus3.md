## 1. Резюме аудита

AKLAB — **single-tenant** инструмент для инвестора/риелтора в коммерческую недвижимость,
приобретаемую на торгах (банкротство, приватизация, маркетплейсы). Сервис парсит 10+ площадок,
считает отклонение цены от эталона (`focus_score`), ведёт объекты по статусам и рассылает
утренний email-дайджест.

**Вердикт:** зрелый, аккуратно структурированный проект с сильным фронтенд-каркасом
(тонкие views, композаблы, ui-kit, тема, задел по a11y). Основные проблемы —
не в архитектуре, а в **деталях UX**: обратная связь при ошибках «проглатывается» в консоль,
несколько функциональных UX-багов (поиск, геокодинг, CIAN-линк) и пробелы в доступности.
Отдельным блоком — точечные проблемы безопасности и корректности на бэкенде.

## 2. Портрет пользователя и карта экранов

**Пользователь:** один инвестор в коммерческую недвижимость. Цель — быстро находить
объекты дешевле рынка и вести их до сделки.

**Сценарий (happy path):**

1. **Дашборд** — 5 KPI (всего / в фокусе / горячие ≥50 / недооценённые / новые 24ч),
   бар-чарт по типам, топ-5 горячих, мини-виджет статуса парсеров.
2. **Объекты** — табы `Все` / `В фокусе` / `В работе`. Запуск pipeline (парсинг→анализ→дайджест).
3. **В фокусе** — поиск, фильтры (порог, город, тип, теги, цена), карточки↔таблица,
   мультивыбор + bulk-действия (статус/CSV), пагинация, экспорт CSV.
4. **Карточка объекта** — hero (цена, бейджи, скор), фотогалерея + lightbox, ссылки
   (источник, «соседи на ЦИАН»), сворачиваемые детали, торги, смена статуса, комментарии, история событий.
5. **Настройки** — правила парсинга, эталоны цен, источники (health/cron), настройки дайджеста.

## 3. Оценка UI/UX (основной блок)

### 3.1 Сильные стороны
- **Чистая архитектура фронтенда:** тонкие views, логика в композаблах
  (`usePropertyData`, `useFocusTab`), переиспользуемый ui-kit (`BaseCard/Badge/Button/Input`,
  `StatCard`, `ToastHost`, `Icon`).
- **Тема dark/light** через CSS-переменные — консистентно во всех экранах.
- **Задел по a11y:** skip-link, `aria-label`, `role="navigation"`, `aria-live`, focus-стили.
- **Адаптивность:** мобильное меню (гамбургер), сворачиваемые фильтры, скелетоны загрузки,
  переключение карточки↔таблица с сохранением выбора в `localStorage`.

## 4. Оценка архитектуры, безопасности и кода (backend)

Компактный блок — вторичный по отношению к UI, но содержит важные пункты.

| # | Severity | Файл | Проблема | Рекомендация |
|---|----------|------|----------|--------------|
| B1 | 🔴 Безопасность | `api/.../routes/property.ts` | `/properties/focus`, `/properties/stats`, `/photos/*` c `auth: false` — ядро ценности (недооценённые объекты) публично | API-token авторизация для сервисов (digest) вместо `auth:false`; либо internal-роут на localhost |

| B3 | 🔴 Корректность | `pipeline/stages.ts` (analyze) | `undervaluedCount` считается через `findMany({limit:1}).length` → финал всегда «0/1 недооценённых» | Реальный SQL `COUNT(*)` |
| B4 | 🟠 Производит. | `property/controllers/property.ts` (getStats) | 6 полных сканов таблицы с `.length` в JS вместо агрегатов | `SELECT COUNT(*)` + `GROUP BY property_type` |

## 7. Метрики успеха
- **UX:** 0 «тихих» ошибок — каждый провал сети/действия виден пользователю (toast).
- **UX:** поиск в фокусе возвращает результаты по всей выборке, а не по странице.
- **a11y:** lightbox и модалки проходят проверку клавиатурной навигации; axe-core без критичных нарушений.
- **Безопасность:** ни один endpoint с бизнес-данными не доступен без авторизации.
- **Корректность:** финал pipeline показывает реальное число недооценённых.
- **Производительность:** `/properties/stats` выполняется 1–2 SQL-запросами вместо 6 сканов.
